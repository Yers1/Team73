# MedPrice.kz — агрегатор и сравнение цен на медицинские услуги Казахстана

Кейс «MedServicePrice» (Кейс 1). Команда № 73, Караганда, 2026.

**Живое демо: https://medtech-self.vercel.app**

---

## О проекте

Когда человеку нужно сдать анализ или попасть на приём, он вынужден вручную обходить
десятки сайтов клиник — цены нигде не собраны вместе, и сравнить их негде. В итоге
пациенты переплачивают просто потому, что не знают, где дешевле.

Мы сделали сервис, который собирает прайсы клиник, приводит разные названия услуг к
единому справочнику и за один поиск показывает, где услугу пройти выгоднее. За время
хакатона довели идею до рабочего MVP: парсер, нормализация данных, REST API,
веб-интерфейс — и развернули всё в интернете.

## Что мы сделали

**Сбор данных.** Написали парсер прайс-листов 8 клиник в форматах PDF, DOCX и Excel,
включая отсканированные документы с очисткой ошибок распознавания. Сырые данные
храним отдельно от обработанных, ведём журнал ошибок парсинга, при повторном запуске
не создаём дубликаты.

**Нормализация.** Разные названия одной услуги («ОАК», «общий анализ крови»,
«клинический анализ крови») сводим к единому справочнику. Основной механизм —
сопоставление по официальному коду тарификатора; дополнительно работают синонимы и
нечёткое сравнение названий. Услуги, которые не удалось сопоставить, попадают в
очередь ручной разметки.

**Интерфейс.** Поиск с автодополнением, фильтры по городу, категории и цене,
сравнение цен по клиникам с отклонением от медианы рынка, карточка клиники и дата
обновления цены. Интерфейс адаптивный, доступен на русском и казахском языках.

**Дополнительные возможности.** Оптимизатор корзины услуг (подбирает, где пройти
набор анализов дешевле всего, раскладывая по клиникам), история цен по годам, карта
клиник и ассистент, понимающий запросы на естественном языке.

## Технологии

- **Бэкенд:** Python, FastAPI, SQLite
- **Парсинг:** pdfplumber, python-docx, openpyxl; нечёткое сравнение — rapidfuzz
- **Фронтенд:** React, Vite, Tailwind CSS
- **Развёртывание:** Vercel

## Запуск

Нужны Python 3.11+ и Node.js 18+.

### Бэкенд
```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --port 8000
```
API: http://127.0.0.1:8000 · документация: http://127.0.0.1:8000/docs

База данных уже собрана и лежит в `backend/data/medservice.db` — для запуска
перепарсинг не нужен. Чтобы пересобрать базу из исходников
(`python -m pipeline.run_pipeline`), нужно положить в корень прайс-файлы клиник
(они предоставлены организаторами и в репозиторий не включены).

### Фронтенд
```bash
cd frontend
npm install
npm run dev
```
Открыть http://127.0.0.1:5173

## Структура

```
backend/
  pipeline/     парсеры, нормализация, ETL (config, db, parsers, normalize, run_pipeline)
  app/          FastAPI (main.py)
  data/         medservice.db — база с данными
api/            точка входа для Vercel
frontend/       React-приложение (Vite + Tailwind)
deliverables/   база данных и справочник услуг отдельными файлами (csv/xlsx)
docs/           скриншоты интерфейса
```

## Ассистент и язык

Ассистент понимает запросы и симптомы на русском и казахском и отвечает на основе
данных из базы. По умолчанию работает на нашем алгоритме без внешних сервисов;
при необходимости можно подключить бесплатный LLM, задав переменную окружения
`GROQ_API_KEY` или `GEMINI_API_KEY` перед запуском бэкенда.

## Ограничения и данные

Используются только открытые публичные прайсы клиник. Персональные данные пациентов
не собираются. У каждой цены сохраняется источник и дата сбора.
